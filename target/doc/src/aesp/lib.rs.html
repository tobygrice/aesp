1//! This crate provides an intuitive interface for AES-128, AES-192, and AES-256 encryption and decryption.
2//! The following modes of operation are supported:
3//! - **Galois/counter mode** ([GCM](crate::Cipher::encrypt_gcm)), with optional additional authenticated data (AAD). 
4//! Encrypts using CTR mode and generates an authentication tag from the AAD + ciphertext. This tag is recomputed at decryption 
5//! and compared with the received tag.
6//! - **Counter mode** ([CTR](crate::Cipher::encrypt_ctr)). A 16-byte counter is repeatedly incremented and encrypted. 
7//! The result is `XOR`'d with the plaintext to produce the ciphertext. 
8//! This turns AES into a stream cipher, which removes vulnerabilities present in modes such as ECB.  
9//! - **Electronic codebook mode** ([ECB](crate::Cipher::encrypt_ecb)). Encrypts each block of plaintext seperately and appends to the output. 
10//! Vulnerable to pattern emergence in larger inputs. Use a stream cipher mode (CTR or GCM) if security is important. 
11//! 
12//! ## Examples
13//! Below is an example of a string being encrypted under a random key using AES-256-CTR, then decrypted back to plaintext.
14//! ```
15//! # fn main() -> aesp::Result<()> {
16//! use aesp::{Key, Cipher};
17//! 
18//! // generate a random 256-bit key.
19//! let key = Key::rand_key_256()?;
20//! 
21//! // instantiate a cipher object using that key.
22//! let cipher = Cipher::new(&key);
23//! 
24//! // instantiate sample plaintext (cipher encrypts raw bytes).
25//! let plaintext = ("Hello, World!").as_bytes();
26//! 
27//! // encrypt the plaintext bytes using AES-256-CTR.
28//! // note that the key size does not need to be explicitly stated.
29//! let ciphertext = cipher.encrypt_ctr(&plaintext)?;
30//! 
31//! // decrypt the resultant ciphertext.
32//! let decrypted_ct = cipher.decrypt_ctr(&ciphertext)?;
33//! 
34//! // round trip results in the same plaintext as the original message.
35//! assert_eq!(plaintext, decrypted_ct);
36//! 
37//! # Ok(())
38//! # }
39//! ```
